# polyagents 架构说明

> 更新: 2026-06-09 · 仓库: github.com/yuanziding123-bot/polyagent · 测试: 83 passed
> 配套: [项目状态与待决策.md](项目状态与待决策.md) · [README.md](README.md)

polyagents 有**两个视角**:
1. **交易引擎** —— 一条 LangGraph 四层流水线(数据→决策→执行→反馈)
2. **平台能力** —— 把引擎打包成 **skills + MCP**,供 Alpha DevBox 式聊天平台加载

---

## 一、总览(双视角)

```
┌──────────────────────────── 平台层(Alpha DevBox 外壳)───────────────────────────┐
│  用户聊天  ──►  Claude Agent  ──加载──►  skills/*.SKILL.md   (工作流知识)         │
│                      │        ──连接──►  polyagents MCP server (工具)             │
│                      ▼                                                             │
│   ┌──────────────────────────── polyagents 交易引擎 ───────────────────────────┐ │
│   │   L1 数据收集 ──► L2 决策引擎 ──► L3 执行 ──► L4 反馈闭环                    │ │
│   │       │                                          ▲                          │ │
│   │   SQLite 库 + ChromaDB RAG + Memory(JSONL)  ────┘ (教训回灌)               │ │
│   └─────────────────────────────────────────────────────────────────────────────┘ │
└────────────────────────────────────────────────────────────────────────────────┘
```

- **直接用引擎**:`PolyAgentsGraph().collect/analyze/trade/settle()`(Python)
- **平台化用**:宿主 Agent 通过 **MCP 工具** + **SKILL.md** 驱动(聊天即用)

## 二、交易引擎:四层流水线(核心)

一条 LangGraph `StateGraph`,一个共享的**黑板状态 `MarketState`** 流经所有节点,每个节点读它、写部分更新。

```
START
  │
  ▼   ════════════ L1 数据收集(确定性 · 0 token)════════════
  market_data ─► orderbook ─► trades_flow ─► news ─► features
   价格/量能      L2微结构     买卖流向      情感    因子向量(汇总)
   (官方SDK)    (micro-price  (flow         (词典   (Alpha DevBox
               /depth/压力)   imbalance)    占位)   式 join)
  │
  ▼   ════════════ L2 决策引擎(三 Agent)════════════
  signal ──────────► decision ───────────► reflection
  (LLM)              (确定性数学)            (LLM)
  估 p_true          edge=p_true−price       交易前自检
  追踪资金           1/4 Kelly + 风控闸门    风险标记
                     (流动性/点差/6%门槛)
  │
  ▼   ════════════ L3 执行 ════════════
  execute ─► 端口/适配器:PaperExecutionClient(默认) / LiveCLOB(gated)
            ─► 熔断器(日亏/敞口/并发/连亏)─► Portfolio 记账
  │
  ▼  END

  ════════════ L4 反馈闭环(市场结算后,异步触发 settle())════════════
  settle() ─► 按 token 判胜负 ─► 纸面 $1/$0 兑付 ─► 已实现 P&L 入账
                   │                                      │
                   └─► reflect_on_outcome(LLM) ─► Lesson ─┘
                                  │
                                  └──注入──► 下次 signal 的 prompt(carry-forward 学习)
                                  └──标注──► RAG 市场向量(打上赢家标签)
```

**入口对照**

| 方法 | 跑到哪层 | 是否要 key |
|---|---|---|
| `collect(market)` | L1 | ❌ 免 key |
| `analyze(market)` | L1+L2 | ✅ 信号/反思要 Anthropic key |
| `trade(market)` | L1+L2+L3 | ✅ 同上(执行是纸面/确定性) |
| `settle()` / `report()` | L4 | settle 默认要 key(可 `reflect=False` 关掉) |

## 三、黑板状态(数据怎么流)

不是消息传递,是**黑板**:节点把结果写进 `MarketState` 的字段,后面的节点直接读。

```
                ┌──────────────── MarketState(黑板)────────────────┐
                │ 身份: market_id/condition_id/token_id/question…   │
   各节点 ──读/写─►│ L1 报告: price/volume/orderbook/trades_flow/news │
                │ raw{}:   price/volume/orderbook/trades_flow/      │
                │          news/features  ← 结构化数字              │
                │ L2: signal / trade_decision / reflection          │
                │ L3: execution_result                              │
                └───────────────────────────────────────────────────┘
```
> 设计来自 TradingAgents 的 `AgentState`(继承 `MessagesState`)。`raw` 既存"给人/LLM 看的报告",也存"给数学/ML 用的数字"。

## 四、持久化三件套

```
~/.polyagents/
  cache/polyagents.db   (SQLite)  markets / candles / trades(+水位线缓存) /
                                   orderbook_snapshots / collections(整次因子包)
  cache/chroma/         (Chroma)  市场向量库 → 相似市场检索(RAG)
  memory/trades.jsonl   (JSONL)   决策日志 + 结算结果 + 教训(L4 记忆)
```
- **SQLite**:缓存(成交不重复翻页)+ 历史积累(给 ML/回测)
- **Chroma**:语义检索"相似历史市场 + 怎么结算的"
- **JSONL Memory**:决策→结果→教训,教训回灌下次决策

## 五、平台化:skills + MCP

只改 polyagents,Alpha DevBox 当外壳。**宿主 Agent 负责推理,polyagents 提供确定性工具。**

```
┌─ Alpha DevBox(Node/Docker 聊天外壳,不改它)
│    │ 用户: "扫一下 Polymarket 找机会"
│    ▼
│  Claude Agent
│    ├─ 加载 skills/polymarket-trading/SKILL.md   (告诉它工作流 + 纪律)
│    └─ 连接 polyagents MCP server               (拿到工具)
│         │
└─────────┼──────────────────────────────────────────────────────
          ▼  polyagents/mcp_server.py (FastMCP · 确定性 · 免 key)
   scan_markets ─► market_snapshot ─► find_similar_markets
        └─► size_position(Kelly+风控)─► paper_execute(过熔断)
        └─► portfolio_status / settle_markets / pnl_report
          │
          ▼  调用 L1–L4 引擎 + SQLite/Chroma/Memory
```

**一次对话的典型流转**
```
Agent: scan_markets()            → 拿到候选(token_id…)
Agent: market_snapshot(tok)      → 看微结构/流向/因子
Agent:(自己推理)                → 估 p_true = 0.62
Agent: size_position(0.62, tok)  → 返回 {action: buy, edge, size_usdc} 或 hold
Agent: paper_execute(tok,buy,$)  → 纸面成交,更新组合
…市场结算后…
Agent: settle_markets()          → 兑付 + 已实现 P&L
Agent: pnl_report()              → 命中率/盈亏归因
```

**加新 skill = 两步**:在 `mcp_server.py` 加几个 `@mcp.tool()` + 写一个 `skills/<name>/SKILL.md`。外壳零改动。

## 七、几条贯穿全局的设计原则

1. **黑板模式**(来自 TradingAgents):单一 `MarketState` 流经所有节点,易扩展、易断点。
2. **确定性优先,LLM 最后**:只有 signal/reflection/结算反思 3 处花 token;数据/微结构/因子/Kelly/执行/结算全是免费数学。→ 省 token + 可审计。
3. **可插拔 seam**:`PolyAgentsGraph(scorer=…, forecaster=…, llm=…, execution_client=…)` 全可注入;LangGraph 节点可组装;MCP 工具/技能可增删。→ 这正是"平台化"的地基。
4. **纸面/只读默认**:不配 key、不下真单;实盘要显式开 `execution_mode="live"` + 私钥。
5. **三档接入开源项目**:能跑就真接(LangGraph/官方SDK/ChromaDB/MCP)、有门槛就建好等资源(MarketLens/qlib)、名不副实就借鉴自建(TradingAgents架构/微结构/因子/情感)。
